🔹 MACHINE LEARNING BASICS

What is Machine Learning?
Machine Learning is a subset of artificial intelligence that enables systems to learn and improve from experience without being explicitly programmed. ML algorithms build mathematical models based on training data to make predictions or decisions.

Types of Machine Learning:

1. Supervised Learning
The algorithm learns from labeled training data. Input and output are both provided. Examples include classification (spam detection) and regression (price prediction). Common algorithms: Linear Regression, Logistic Regression, Decision Trees, Random Forest, Support Vector Machines (SVM).

2. Unsupervised Learning
The algorithm works on unlabeled data and finds hidden patterns or intrinsic structures. Examples include customer segmentation and anomaly detection. Common algorithms: K-Means Clustering, Hierarchical Clustering, Principal Component Analysis (PCA), DBSCAN.

3. Reinforcement Learning
The algorithm learns by interacting with an environment, receiving rewards or penalties. Used in robotics, game playing, and self-driving cars. Examples: Q-Learning, Deep Q Networks (DQN), Policy Gradients.

Machine Learning Lifecycle:

Step 1: Problem Definition - Understand business problem and define success metrics
Step 2: Data Collection - Gather relevant data from databases, APIs, or web scraping
Step 3: Data Preprocessing - Handle missing values, remove duplicates, normalize data
Step 4: Exploratory Data Analysis - Visualize data, find correlations, identify patterns
Step 5: Feature Engineering - Create new features, select important features
Step 6: Model Selection - Choose appropriate algorithm based on problem type
Step 7: Model Training - Split data into training and testing sets, train the model
Step 8: Model Evaluation - Measure performance using accuracy, precision, recall, F1 score
Step 9: Hyperparameter Tuning - Optimize model parameters for better performance
Step 10: Model Deployment - Deploy model to production for real-world predictions

Key Metrics for Evaluation:

Classification Metrics:
- Accuracy = (TP + TN) / (TP + TN + FP + FN)
- Precision = TP / (TP + FP)
- Recall = TP / (TP + FN)
- F1 Score = 2 * (Precision * Recall) / (Precision + Recall)

Regression Metrics:
- Mean Absolute Error (MAE)
- Mean Squared Error (MSE)
- Root Mean Squared Error (RMSE)
- R-Squared (R²)

Overfitting and Underfitting:

Overfitting occurs when a model learns training data too well, including noise, but fails on new data. Solutions: more training data, reduce model complexity, regularization, cross-validation.

Underfitting occurs when a model is too simple to capture patterns in data. Solutions: increase model complexity, add more features, train for longer.

Bias-Variance Tradeoff:
- High Bias: Model is too simple (underfitting)
- High Variance: Model is too complex (overfitting)
- Goal: Find the sweet spot between bias and variance

Common Interview Questions:

Q1: What is the difference between supervised and unsupervised learning?
A1: Supervised learning uses labeled data with input-output pairs. Unsupervised learning uses unlabeled data to find hidden patterns.

Q2: Explain the bias-variance tradeoff.
A2: Bias is error from wrong assumptions. Variance is error from sensitivity to training data. Simple models have high bias. Complex models have high variance.

Q3: What is cross-validation?
A3: Cross-validation is a technique to evaluate model performance by splitting data into multiple training and validation sets.

Q4: How do you handle missing data?
A4: Options include removing rows, filling with mean/median/mode, or using algorithms that handle missing values.

Q5: What is regularization?
A5: Regularization adds penalty to complex models to prevent overfitting. L1 (Lasso) and L2 (Ridge) are common types.
